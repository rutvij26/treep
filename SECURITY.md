# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Treep, please report it responsibly:

1. **Do not** open a public GitHub issue
2. Email the maintainer: [Your email or security contact]
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Security Response

- Reports will be acknowledged within 48 hours
- We will work to provide a fix within 30 days
- Security updates will be released as patch versions
- Vulnerabilities will be disclosed after a fix is available

## Security Best Practices

When using Treep:

- Keep dependencies up to date
- Validate all input data using schema validation
- Be cautious when processing untrusted data
- Review error messages to avoid information leakage

## Known Security Considerations

- Treep processes user-provided data structures
- Large graphs may consume significant memory
- Always validate input data before processing
